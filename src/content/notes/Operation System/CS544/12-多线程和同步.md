# 并发

其原因是性能，CPU 的速度收到频率和功率限制，在 2005 年以后，开始以非线性速度增长。

因此，人们开始使用相同的时钟速度，更多的 CPU 核心，让这些核心同时执行任务会增加系统的性能。

# 进程和线程

此时有两种并发的选择，进程和线程。

## 进程

每一个执行都运行在隔离的环境中，不共享内存空间，如果需要共享，则使用进程间通信，比如 `File(), Pipe(), socket(), shared memory`。

![image](/notes-assets/6ec611e8/image-20231128171209760.png)

- 优点
	- 无需更改程序
- 缺点
	- 使用额外内存
	- 无法直接访问其他进程的内存，需要进程间通信
- 使用
	- 适合并行隔离的执行
	- 不适合并行共享数据的执行

## 线程

和进程不同，线程可以共享内存空间。传统的方法是使用虚拟内存映射或者使用只读并在需要写入时创建新内存（Copy-on-Write）。而进程只创建一个新的执行空间（new execution environment），使用同一个页目录和表 (same page directory or table)，但是创建一个新的栈用于存储寄存器 (new stack for register context)。

![image](/notes-assets/6ec611e8/image-20231128171824749.png)

- 优点
	- 访问同一片内存空间
	- 更少的内存需求
- 缺点
	- 没有运行隔离，需要小心的改程序
- 用途：和进程相反

## 异同

|             | 进程 | 线程 |
| -------------- | ---- | ---- |
| 内存空间       | 不同 | 相同 |
| 执行内存器变量 | 不同 | 不同 |
| 数据隔离       | 有   | 无   |

# 同步与互斥

而解决数据竞争的方法就是互斥、临界区。他们将多个指令视为一个整体，而每次只有一个线程能访问这部分，其他将被禁止。使用互斥锁的部分相当于单线程的，因此我们必须慎重的使用，更少的使用带来更强的性能。

![image](/notes-assets/6ec611e8/image-20231128180259942.png)

# 实现

在单处理器计算机上，我们可以试用`cli`和`sti`指令。`cli` 会清除中断 bit 位，在运行 `sti` 之前计算机不会中断。而`sti`会设置中断位。

而在一般情况，我们希望有`lock()`和`unlock()`函数。lock 将会检查是否有其他的线程使用临界区，如果没有，就请求锁，如果有，就等待。临界区执行完毕后，unlock 将会释放锁。

![image](/notes-assets/6ec611e8/image-20231128185534799.png)

## SpinLock

会使用循环检查 critical section 是否被占用。

```c
// lock
while (*lock == 1);
*lock = 1;
// unlock
*lock = 0;
```

但是问题在于，存储和读取的过程本来就是 race condition 的温床。因此，我们需要硬件上指令的支持，来一次性存储寄存器中的数据到内存。

## 原子数据交换

```c
xchg [memory], %reg
```

xchg 的作用是将寄存器中的数据。
