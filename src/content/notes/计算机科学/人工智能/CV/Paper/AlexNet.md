# 总结

总体流程
- 深度神经网络可 End-to-End，无需 SIFT 特征提取
- 没有无监督的预训练
- CNN 需要大数据集，强力 GPU，否则过拟合

网络架构
- 深度网络比浅层好
- 调参和高宽比都很重要
- 在卷积的过程中，不断把少通道的高宽扁信息变化为多通道的压缩信息
- 最后一层的输出在语义空间中表示很好（可以作分类器）

优化方法
- 激活函数 ReLu：$f(x) = max(0, x)$，单侧抑制，简单有效，降低过拟合
- 数据增强
	- 图片变化：随机从 256 扣 224
	- Dropout：把神经元变 0，等同于 L2 正则，对很大的全连接有用（RNN 和 Attention）

训练方法
- SGD，参数难调但是其中的噪音对泛化性有好处
- momentum：一般 0.9，主要用于优化表面不平滑的时候，冲量使梯度不要被当下的梯度过多的误导，可以沿着一个比较平缓的方向向前走，不容易局部最优
- weight decay 是 0.0005，也就是 L2 正则的权重惩罚
- standard deviation 是 0.01，综合的比较好的值，深度网络需要更多优化，但是后来的 bert 也就 0.02。
- bias 初始化：一般初始化为 0 效果就不错。本文一部分 0 一部分 1。
- learning rate：先 warmup 上升，然后 cos LR 下降（ResNet：每 30 轮下降 10 倍）

## 架构

主要的贡献之一。

### ReLu

资料：[ReLU 激活函数中神经元死亡问题](https://murphypei.github.io/blog/2020/09/relu-dying)

非线性激活函数，$f(x) = max(0, x)$，负值全部变为 0，单侧抑制。单侧抑制的存在使得部分神经元可能输出为0，相当于在本次优化中，一些神经元不参与，进而破坏了全部神经元联动优化的陷阱，使得模型更加的鲁棒。但是也存在一个风险。假如某个神经元一直输出负值，则经过激活函数relu后始终输出为0，这就会导致在每一次的反向优化中，改神经元始终不参与，进而导致神经元成为死亡节点。

当时：过去用的 tanh 或者 sigmoid，这些饱和的非线性会慢。

现在：其实也没快多少，主要是简单

- 四层：各个的
- 五层：各个的（通道数不断增加，高宽会变化）；输入是一个很扁的图片，高宽慢慢变小，深度慢慢增加；压缩空间信息到 13\*13，增加通道数（看特定的模式，语义增加）

4096 的向量可以很好的表示图片的信息，压缩原始数据。

深度神经网络的精髓：人看懂的像素，通过特征提取，压缩变成机器能懂的向量。

过于复杂的技术细节，强行分割成两个，后面看用处不大。但是最近由于大模型的出现，在 NLP 里面很主流了。

## 降低过拟合

1. 数据增强
	1. 人工把图片变大：随机扣 224 从 256，有 2048 种扣法
	2. 颜色通道改变
2. dropout
	1. 把其中一些神经元便 0，变成一个新的模型。但后来发现其等价于 L2 正则。
	2. 训练慢两倍
	3. 对很大的全连接很有用
	4. 现在 CNN 用的不多，因为不使用很大的全连接（这是当时设计的缺陷）；RNN 和 Attention 用的很多

## 训练细节

用 SGD 训练，当时不流行，后来成为了主流算法

- 参数很难调
- 里面的噪音对泛化性有好处

参数选择

- batch size 是 128。
- momentum 是 0.9
	- 虽然 10 年左右有很多SGD 加速算法，但是现在看来一个很简单的 momentum 也不错。
	- 优化的表面不平滑的时候，冲量使梯度不要被当下的梯度过多的误导，可以沿着一个比较平缓的方向向前走，不容易局部最优
- weight decay 是 0.0005，也就是 L2 正则的权重惩罚
- standard deviation 是 0.01，综合的比较好的值，深度网络需要更多优化，但是后来的 bert 也就 0.02。
- bias 初始化：一部分是 0，另一部分是 1
	- 后来不用，一般初始化为 0 效果就不错。
- 学习率：从 0.01 开始，验证误差不下降了，就降低 10 倍
	- ResNet：每 30 轮下降 10 倍
	- 后来用更平滑的 cos
	- 现在先 warmup 上升，然后 cos LR 下降

## 结果

主要在说结果

发现了 GPU1 上学习到的是和颜色无关的，GPU2 的是和颜色有关的。但后来这块没人跟进了。

相似图片的最后一层卷积的输出很类似（L2 距离小），即使从像素角度看，他们的差距很大。

现在：底层学局部信息（纹理、方向）；上层学全局（头、手、动物）。但是依旧是黑箱，可解释性差。
